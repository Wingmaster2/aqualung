Project policies

Notices about smaller or greater development advancements, CVS commits, etc. will go to our catch-all mailing list called aqualung-friends. This list should serve as the primary forum of development discussion, as well as user support. If you have a question regarding the compilation or usage of Aqualung, please join the list and ask there. We will discuss new features and design decisions there as well, and you are hereby invited to take part in this process. Finally, if you have patches, they should be sent to the list as well. If not accepted immediately, your contribution will be discussed in an open way.

Bugs should be reported primarily to our bugtracker. Feel free to add your feature requests there as well (use the appropriate category). You can also report bugs directly to the mailing list, but the preferred way is to use the bugtracker. If you are not sure if you found a real bug, feel free to write to the list prior to filing a bugreport. But it's OK to report a non-bug into the bugtracker, so don't be too shy.

Announcements of major advancements of Aqualung development will be CC:-ed to the Linux Audio Developers, the Linux Audio Users and the Linux Audio Announce mailing lists.

Translators

German, Hungarian:Peter Szilagyi <peterszilagyi at users dot sourceforge dot net>
Ukrainian:Sergiy Niskorodov <sgh_punk at users dot sourceforge dot net>

New translations for any language are happily accepted at any time. Just roll the .po file for your language based on the master src/po/aqualung.pot file, and send it to the mailing list. We'll include it in the distribution.

As Aqualung development progresses, some text changes and new strings appear. Translations therefore need to be updated from time to time. If you are a translator, send an updated version of your .po file to the mailing list at any time you feel fit.

Thank you all!

Thanks to everyone who contributed to Aqualung's development, be it patches, comments, suggestions, bugreports, feature requests, etc. You may not even realize that your letter asking for help with compilation/usage/etc. gave us ideas to make Aqualung "Software That Sucks Less(TM)". Thanks to everyone who downloaded and tried the program.

Aqualung could not be functional without the following libraries. We would like to thank the authors of these software packages:

  • The GIMP Toolkit (GTK+), version 2.0
  • libXML2, the XML C parser and toolkit of Gnome
  • libJACK, the JACK Audio Connection Kit support library
  • liblrdf, a library for manipulating RDF files about LADSPA plugins
  • libsamplerate (aka Secret Rabbit Code), a high quality Sample Rate Converter
  • libsndfile, a library for accessing audio files containing sampled sound
  • libvorbis & libvorbisfile, for decoding Ogg Vorbis audio
  • libFLAC, the Free Lossless Audio Codec library
  • libMAD, a GPL'ed MPEG Audio Decoder library
  • libmodplug, a MOD decoder library in the public domain
  • libmusepack, a GPL'ed Musepack decoder library
  • libid3tag, a library for accessing ID3v2 tags
